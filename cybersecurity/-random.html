Device to Server (networking)

Device → Switch/Access Point → Router → Modem → ISP Server 1 → Internet Backbone → ISP Server 2 → Server

Device

Physical device that connects to the network.

Switch

Connects to devices via physical ethernet (wired).

Access Point (AP)

Connects to devices via WI-FI (wireless).

Router

  • "Manages" a network
  • Connects LAN (Switch/Access Point) and WAN (Modem)
  • DNS Server query

Modem

Connects to your Internet Service Provider (ISP) server.

ISP Server 1

Connects from your modem to the "internet backbone."

Internet Backbone

It essentially acts as a highway allowing data to travel quickly, think of the ISP servers as exits/entrances to the highway. Connects from ISP Server 1 to ISP server 2.

ISP Server 2

Connects from the "internet backbone" to the server you wanted to reach.

and in reverse to get the data back... The whole process is called "routing", which is a process that allows data to travel from one place to another in the most efficient pathway.

Command Prompt vs Windows Powershell

Command Prompt

  • Scripting file format: .bat
  • A text-based command-line interpreter in Windows.
  • Executes commands through simple executables and built-in commands.
  • Text-based input/output.
  • Limited scripting capabilities.

PowerShell (pwsh):

  • Script file format: .ps1
  • A command-line shell and scripting language built on the .NET framework.
  • Executes commands through cmdlets (verb-noun commands).
  • Designed for system administration, automation, and advanced scripting.
  • Object-based input/output.
  • Powerful scripting features and .NET integration.
  • Can run cmd.exe commands and executables.

TL;DR

command prompt (engine: cmd.exe) = basic
PowerShell (engine: pwsh.exe or powershell.exe)= powerful.
PowerShell is the future.
